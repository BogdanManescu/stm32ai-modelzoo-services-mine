     Plan: Synthetic States Dataset Generator + Training Config

     Context

     The user needs to train an AI model that detects 8 human-condition states
     from a 16×128 pixel image (16 tall, 128 wide) divided into 8 blocks of
     16×16 pixels. Each block represents a state: active = pure white
     (255,255,255), inactive = pure black (0,0,0). States 0-6 can be active in
     any combination (2^7 = 128 combos). State 7 ("Normal") is active ONLY when
     all states 0-6 are inactive. Since image classification outputs a single
     class, each combination maps to a unique class — 129 classes total.

     The training data is clean: pure black/white blocks with no noise or
     variations, matching how real sensor data will be generated.

     Design Decisions

     - 129 classes: 128 combinations of states 0-6 + 1 "normal" (only block 7
     white)
     - Class naming: s0_s2_s4 for active states, none when no states 0-6 are
     active, normal for state 7 only
     - Image format: 16×128 RGB PNG, pure black (0,0,0) and white (255,255,255)
     pixels only
     - Model: st_mnistv1 (~6K params, no input shape constraints)
     - Input shape: (16, 128, 3) — images generated at target size, no resizing
     needed
     - Framework: TensorFlow

     Files to Create

     1. image_classification/generate_states_dataset.py

     Dataset generation script with these features:

     - Image: 16×128 RGB PNG, divided into 8 blocks of 16×16 pixels (block N:
     columns [N*16, (N+1)*16-1])
     - No variations — pure black (0,0,0) and white (255,255,255) only
     - CLI args: --train-count N, --val-count N, --test-count N, --output-dir 
     PATH, --seed N
     - Output structure:
     output_dir/
       labels.txt
       train/
         s0/
         s0_s1/
         ...
         none/
         normal/
       val/   (same structure)
       test/  (same structure)
     - labels.txt: 129 lines, alphabetically sorted class names

     2. image_classification/config_file_examples/training_states_config.yaml

     Training config wired for this task:

     - operation_mode: training
     - model.model_name: st_mnistv1, input_shape: (16, 128, 3), pretrained: 
     False
     - dataset.dataset_name: custom_dataset, classes_file_path: pointing to
     labels.txt
     - preprocessing.rescaling: {scale: 1/255.0, offset: 0}, resizing to
     (16, 128)
     - Minimal data augmentation only (random brightness maybe useful)
     - training.epochs: 50, batch_size: 32, dropout: 0.2

     Verification

     1. Generate dataset: python generate_states_dataset.py --train-count 200 
     --val-count 50 --test-count 50 --output-dir ./datasets/states_dataset
     2. Check directory structure and spot-check a few PNGs
     3. Run training: python stm32ai_main.py --config-path 
     ./config_file_examples/ --config-name training_states_config.yaml
     4. Verify training starts and loss decreases

  Option B: Define your own model

  Two steps:

  Step 1 — Create your model file at tf/src/models/my_states_model.py:

  import keras
  from keras import layers

  def get_my_states_model(num_classes=None, input_shape=None, dropout=None,
                          pretrained=False, **kwargs) -> keras.Model:
      inputs = keras.Input(shape=input_shape)

      # Small conv stack — keep strides low since input is only 16px tall
      x = layers.Conv2D(32, 3, padding='same')(inputs)
      x = layers.BatchNormalization()(x)
      x = layers.ReLU()(x)
      x = layers.MaxPooling2D(2)(x)  # 16→8

      x = layers.Conv2D(64, 3, padding='same')(x)
      x = layers.BatchNormalization()(x)
      x = layers.ReLU()(x)
      x = layers.MaxPooling2D(2)(x)  # 8→4

      x = layers.Conv2D(128, 3, padding='same')(x)
      x = layers.BatchNormalization()(x)
      x = layers.ReLU()(x)

      x = layers.GlobalAveragePooling2D()(x)
      if dropout:
          x = layers.Dropout(dropout)(x)
      x = layers.Dense(num_classes, activation='softmax')(x)

      return keras.Model(inputs, x, name="my_states_model")

  Step 2 — Register it in tf/wrappers/models/custom_models/custom_models.py. Add
   one line to the TF_CUSTOM_MODEL_FNS dict:

  TF_CUSTOM_MODEL_FNS = {
      'custom_model':         (get_custom_model, {}),
      'st_efficientnetlcv1':  (get_st_efficientnetlcv1, {}),
      'st_fdmobilenetv1':     (get_st_fdmobilenetv1,    {}),

● For this task, st_mnistv1 is the best choice — and it already fits by a huge
  margin. Let me break down the numbers:

  Model size vs. NPU limits

  ┌───────────────────────────────┬───────────────────────────────────┐
  │             What              │               Size                │
  ├───────────────────────────────┼───────────────────────────────────┤
  │ st_mnistv1 (float32)          │ ~68 KB weights                    │
  ├───────────────────────────────┼───────────────────────────────────┤
  │ st_mnistv1 (int8 quantized)   │ ~17 KB weights                    │
  ├───────────────────────────────┼───────────────────────────────────┤
  │ custom_model (int8 quantized) │ ~32 KB weights                    │
  ├───────────────────────────────┼───────────────────────────────────┤
  │ NPU weight memory             │ 4 × 448 KB = 1,792 KB             │
  ├───────────────────────────────┼───────────────────────────────────┤
  │ PU cache                      │ 256 KB (activations/feature maps) │
  └───────────────────────────────┴───────────────────────────────────┘

  Even in float32, st_mnistv1 uses only ~4% of one weight bank.
  Post-quantization (which the deployment pipeline does automatically), it's
  under 1%. The PU cache is also fine — the largest activation map for a 16×128
  input through this model is only ~16 KB.

  Why st_mnistv1 specifically

  The task is binary block pattern detection on 16×128 images — this is
  essentially MNIST-level complexity. st_mnistv1 was designed for exactly this
  kind of simple pattern recognition. It only has 3 conv blocks with 2×
  downsampling, so it doesn't overthink the problem. A bigger model would just
  waste NPU cycles.

  Path to deployment on STM32N6

  The pipeline already handles this — you'd use the chain_qd operation mode
  (quantize → deploy):

  operation_mode: chain_qd

  This runs the model through STEdgeAI Core, which quantizes to int8 and
  compiles it for the NPU. The output is a C project you flash to the N6.

  Stick with st_mnistv1. There's no benefit to a custom model here — the
  existing one is already near the theoretical minimum size for a CNN that can
  do this task.
